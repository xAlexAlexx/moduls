---------fail2ban для защиты ssh------------ 

apt-get install fail2ban python3-module-systemd -y


sed -i 's/before = paths-altlinux.conf/before = paths-altlinux-systemd.conf/' /etc/fail2ban/jail.conf

+++
cat << EOF > /etc/fail2ban/jail.d/sshd.conf
[sshd]
enabled = true
port = 2026
filter = sshd
backend = systemd
maxretry = 3
bantime = 1m
EOF
+++

systemctl enable --now fail2ban
fail2ban-client status sshd

В графике:
1. Создаём организацию irpo:
Заходим в настройки, учетки и создаем организацию
Заходим в организацию и добавляем пользователя irpoadmin

2. Создаём хранилице на узле хранения (Создается во время планов смотреть пункт 3)

3. Создаём планы хранения:
Заходим планы и создаем план, добавляем сервер, указываем место хранения - узел хранения hq-cli, заходим в указания пути, создаем папку backup, выбираем и нажимаем готово.
Задаем имя узла хранения backup_dir, готово.
В выборе данных устанавливаем "файлы/папки", указываем каталог /etc/.
Даём имя плану защиты

Создаем 2 план хранения, указываем место хранения backup_dir, заходим в "Резервное копирование приложения" и включаем MySQL.
Даём имя плану защиты.
Далее устанавливаем галочку на этом плане и заходим в "изменить", добавляем устройство hq-srv и сохраняем.

Далее заходим в "устройства" и создаем резервные копии
control mysqld server
systemctl restart mysqld
